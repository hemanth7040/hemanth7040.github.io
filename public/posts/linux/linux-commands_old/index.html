Linux Commands Every DevOps Engineer Must Know

This is a living reference page. Commands are grouped by category so you can jump to what you need. Bookmark it.

File & Directory Management
#

Working with files is the foundation of everything in Linux.

# List files with full details (permissions, owner, size in human-readable)
ls -lah /var/log

# Recursively find files by name or pattern
find /etc -name "*.conf" -type f
find /var -name "*.log" -mtime -1   # modified in last 24 hours

# Check disk usage of a directory
du -sh /var/lib/docker
du -sh /* 2>/dev/null | sort -rh | head -10   # find largest dirs

# Create, copy, move, delete
mkdir -p /app/config/envs
cp -r ./source ./destination
mv oldname newname
rm -rf /tmp/build-artifacts

# Compress and extract
tar -czf backup.tar.gz ./myapp          # compress
tar -xzf backup.tar.gz -C /opt/myapp   # extract to path
zip -r archive.zip ./folder
unzip archive.zip -d /target

# File permissions and ownership
chmod 755 deploy.sh          # rwxr-xr-x
chmod 600 ~/.ssh/id_rsa      # rw------- (SSH private key)
chown -R appuser:appgroup /var/app

Process & System Monitoring
#

Knowing what’s running (and consuming resources) is critical for debugging and operations.

# Real-time process monitor
top
htop                    # better UI — install with: apt install htop

# List all processes with details
ps aux
ps aux | grep nginx     # filter by process name
ps -ef --forest         # show process tree

# Kill processes
kill -9 1234            # force kill by PID
pkill -f "python app.py"   # kill by matching command name
killall nginx

# Manage systemd services
systemctl status nginx
systemctl start|stop|restart|reload nginx
systemctl enable nginx     # auto-start on boot
systemctl disable nginx

# View service logs (systemd journal)
journalctl -u nginx                         # all logs for nginx
journalctl -u nginx -f                      # follow live logs
journalctl -u nginx --since "2 hours ago"
journalctl -p err -b                        # errors since last boot

# System resource overview
free -h                 # RAM usage
df -h                   # disk space per mount
vmstat 1 5              # CPU, memory, I/O stats every 1s (5 times)
iostat -x 1             # per-disk I/O stats
uptime                  # load averages
uname -r                # kernel version
lscpu                   # CPU info

User & Permission Management
# 

# User management
adduser devops
usermod -aG sudo devops      # add to sudo group
passwd devops
userdel -r olduser           # remove user and home dir

# Switch users
su - devops
sudo -i                      # root shell

# View current user info
whoami
id
groups

# Sudoers (edit with visudo — never edit directly)
sudo visudo

# File permission reference
# r=4, w=2, x=1
# chmod 755 = rwxr-xr-x (owner: rwx, group: r-x, others: r-x)
# chmod 644 = rw-r--r-- (common for config files)
# chmod 600 = rw------- (private keys, secrets)

Networking
#

Essential for debugging connectivity, inspecting ports, and managing DNS in cloud environments.

# Test connectivity
ping -c 4 8.8.8.8
traceroute google.com
mtr google.com              # ping + traceroute combined

# DNS lookup and debugging
nslookup example.com
dig example.com
dig @8.8.8.8 example.com   # use specific DNS server
host example.com

# HTTP requests
curl -I https://api.example.com/health     # headers only
curl -s -o /dev/null -w "%{http_code}" URL # just the status code
curl -X POST -H "Content-Type: application/json" \
  -d '{"key":"value"}' https://api.example.com/endpoint
wget -q -O - https://example.com/script.sh | bash

# Check open ports and listening services
ss -tulnp                  # all listening TCP/UDP with PID
ss -tulnp | grep 8080      # specific port
netstat -an | grep ESTABLISHED
lsof -i :80                # what's using port 80

# Firewall (iptables / ufw)
ufw status
ufw allow 443/tcp
ufw deny 23
iptables -L -n -v --line-numbers

# SSH
ssh -i ~/.ssh/mykey.pem user@10.0.0.5
ssh -L 8080:localhost:8080 user@remote   # port forwarding
scp ./file.conf user@remote:/etc/app/
rsync -avz ./local/ user@remote:/remote/ # sync directories

# Network interfaces
ip addr show
ip route show
ifconfig eth0

Text Processing & Log Analysis
#

The real power of Linux is chaining small tools to process text. This is critical for log analysis in DevOps.

# Search within files
grep "ERROR" app.log
grep -rn "ERROR" /var/log/ --include="*.log"   # recursive with line numbers
grep -v "DEBUG" app.log                        # exclude lines with DEBUG
grep -E "ERROR|WARN" app.log                   # multiple patterns (regex)
grep -c "500" access.log                       # count matches

# Stream and tail logs
tail -f /var/log/syslog               # follow live
tail -f /var/log/nginx/access.log | grep -i "5[0-9][0-9]"   # 5xx errors live
head -n 50 app.log

# awk — column processing
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -20
# (count unique IPs in nginx access log)
awk -F',' 'NR>1 {sum += $3} END {print sum}' data.csv   # sum column 3

# sed — stream editor
sed -i 's/localhost/prod-db.internal/g' config.yml    # in-place replace
sed -n '10,20p' bigfile.log                           # print lines 10-20
sed '/^#/d' config.conf                               # delete comment lines

# sort, uniq, wc
sort -u file.txt              # sort and deduplicate
sort -k2 -n data.txt          # sort by column 2 numerically
uniq -c sorted.txt            # count consecutive duplicates
wc -l file.txt                # line count
wc -c file.txt                # byte count

# cut and paste
cut -d',' -f1,3 data.csv      # extract columns 1 and 3
cut -d':' -f1 /etc/passwd     # extract usernames

# jq — JSON processing (essential for kubectl and API responses)
kubectl get pods -o json | jq '.items[].metadata.name'
kubectl get nodes -o json | jq '.items[] | {name: .metadata.name, cpu: .status.capacity.cpu}'
curl -s https://api.github.com/repos/torvalds/linux | jq '.stargazers_count'

# xargs — pass output as arguments
cat pod-list.txt | xargs kubectl delete pod
find . -name "*.pyc" | xargs rm -f

Docker Commands
# 

# Container lifecycle
docker ps                          # running containers
docker ps -a                       # all containers (including stopped)
docker run -d -p 8080:80 --name web nginx
docker start|stop|restart web
docker rm web
docker rm -f $(docker ps -aq)      # remove all containers

# Images
docker images
docker pull ubuntu:22.04
docker build -t myapp:v1.0 .
docker tag myapp:v1.0 registry.io/myapp:v1.0
docker push registry.io/myapp:v1.0
docker rmi myapp:v1.0

# Inspecting and debugging
docker logs -f --tail=100 web
docker exec -it web bash           # shell into running container
docker exec -it web sh             # if bash not available
docker inspect web
docker stats                       # live resource usage for all containers

# Cleanup
docker system prune -af            # remove everything unused
docker image prune -f              # only unused images
docker volume prune -f

# Docker Compose
docker-compose up -d               # start in background
docker-compose up -d --build       # rebuild before starting
docker-compose down                # stop and remove containers
docker-compose down -v             # also remove volumes
docker-compose logs -f app         # logs for specific service
docker-compose ps
docker-compose exec app bash       # shell into compose service

# Registry login
docker login registry.example.com
cat ~/.docker/config.json          # check stored credentials

Kubernetes (kubectl) Commands
#

As someone working toward MLOps, these are the commands you’ll use every day.

# Context and namespace management
kubectl config get-contexts
kubectl config use-context prod-cluster
kubectl config set-context --current --namespace=ml-team

# Pod operations
kubectl get pods                            # default namespace
kubectl get pods -n kube-system
kubectl get pods -A                         # all namespaces
kubectl get pods -o wide                    # with node and IP info
kubectl get pods -l app=myapp              # filter by label
kubectl describe pod my-pod-xyz            # full details + events
kubectl delete pod my-pod-xyz
kubectl delete pods --all -n staging

# Logs
kubectl logs my-pod
kubectl logs -f my-pod                     # follow
kubectl logs -f my-pod -c sidecar         # specific container
kubectl logs my-pod --previous             # from crashed container
kubectl logs -f deploy/my-app --tail=50

# Exec into pods
kubectl exec -it my-pod -- bash
kubectl exec -it my-pod -c init-container -- sh

# Deployments
kubectl get deployments -A
kubectl describe deployment my-app
kubectl rollout status deploy/my-app
kubectl rollout history deploy/my-app
kubectl rollout undo deploy/my-app         # rollback to previous
kubectl rollout restart deploy/my-app      # rolling restart
kubectl scale deploy/my-app --replicas=5
kubectl set image deploy/my-app app=myimage:v2.0

# Services and networking
kubectl get svc -A
kubectl get endpoints my-service
kubectl port-forward svc/my-service 8080:80   # local port-forward
kubectl port-forward pod/my-pod 5000:5000

# ConfigMaps and Secrets
kubectl get configmaps
kubectl get secrets
kubectl describe secret my-secret
kubectl get secret my-secret -o jsonpath='{.data.password}' | base64 -d

# Apply and manage manifests
kubectl apply -f manifest.yaml
kubectl apply -f manifest.yaml --dry-run=client    # validate only
kubectl apply -f ./k8s/                            # apply whole directory
kubectl delete -f manifest.yaml
kubectl diff -f manifest.yaml                      # show what will change

# Resource usage (requires metrics-server)
kubectl top nodes
kubectl top pods -A
kubectl top pods --sort-by=cpu

# Nodes
kubectl get nodes -o wide
kubectl describe node worker-node-1
kubectl cordon worker-node-1       # prevent new pods scheduling
kubectl uncordon worker-node-1
kubectl drain worker-node-1 --ignore-daemonsets --delete-emptydir-data

# Namespace management
kubectl get namespaces
kubectl create namespace staging
kubectl delete namespace staging

# Quick troubleshooting workflow
kubectl get events -n mynamespace --sort-by='.lastTimestamp'
kubectl get pods -n mynamespace | grep -v Running
kubectl describe pod failing-pod | grep -A 10 Events

Environment Variables & Shell Tricks
# 

# Environment variables
export DB_HOST=prod-db.internal
export DB_PORT=5432
printenv                           # list all
echo $PATH
unset MY_VAR

# .env file loading
export $(cat .env | xargs)
# or with dotenv: source .env

# Aliases (add to ~/.bashrc or ~/.zshrc)
alias k='kubectl'
alias kgp='kubectl get pods'
alias kns='kubectl config set-context --current --namespace'
alias ll='ls -lah'
alias dps='docker ps --format "table {{.Names}}\t{{.Status}}\t{{.Ports}}"'

# Shell history tricks
history | grep kubectl
Ctrl+R            # reverse search history
!!                # repeat last command
!kubectl          # repeat last kubectl command

# Background jobs
long-running-command &         # run in background
jobs                           # list background jobs
fg %1                          # bring job 1 to foreground
nohup command > out.log 2>&1 & # run immune to hangup
screen / tmux                  # persistent terminal sessions

Cron Jobs & Scheduling
# 

# Edit crontab for current user
crontab -e

# Cron syntax: minute hour day month weekday command
# Run at 2:30 AM every day
30 2 * * * /opt/scripts/backup.sh >> /var/log/backup.log 2>&1

# Run every 5 minutes
*/5 * * * * /opt/scripts/healthcheck.sh

# Run every Monday at 9 AM
0 9 * * 1 /opt/scripts/weekly-report.sh

# List all cron jobs
crontab -l
cat /etc/cron.d/*

# Systemd timers (modern alternative to cron)
systemctl list-timers

Performance & Debugging Cheatsheet
#

Symptom Command to Run
High CPU top → press P to sort by CPU; `ps aux –sort=-%cpu
High Memory free -h; `ps aux –sort=-%mem
Disk Full df -h; `du -sh /*
Port in use `ss -tulnp
DNS not resolving dig example.com; cat /etc/resolv.conf
Service won’t start systemctl status svc; journalctl -u svc -n 50
Container crash loop kubectl describe pod; kubectl logs pod --previous
Pod pending kubectl describe pod → check Events for scheduling errors
Node not ready kubectl describe node; check kubelet: journalctl -u kubelet

Quick Reference: Essential One-Liners
# 

# Watch a command every 2 seconds
watch -n 2 kubectl get pods

# Run command on multiple servers
for host in web1 web2 web3; do ssh $host "systemctl status nginx"; done

# Find and kill a process using a port
kill -9 $(lsof -t -i:8080)

# Monitor log in real time, highlight errors
tail -f app.log | grep --color=always -E "ERROR|WARN|$"

# Count lines in all log files
find /var/log -name "*.log" | xargs wc -l | sort -rn | head

# Decode a base64 Kubernetes secret
kubectl get secret mysecret -o jsonpath='{.data.key}' | base64 --decode

# Get all image names running in a cluster
kubectl get pods -A -o jsonpath='{range .items[*]}{.spec.containers[*].image}{"\n"}{end}' | sort -u

# Check SSL certificate expiry
echo | openssl s_client -connect example.com:443 2>/dev/null | openssl x509 -noout -dates
